# 故事骨架

## 输入
一段故事或小说章节文本。

## 输出格式
结构化 JSON，包含以下字段：

- **title**：故事标题
- **logline**：一句话梗概（25 字内）
- **acts**：幕列表，每幕含 name + beats（节拍）
- **episodeCount**：推荐分集数
- **hookPoints**：付费/悬念卡点列表

## 分幕原则
1. 三幕结构：建置（25%）→ 对抗（50%）→ 解决（25%）
2. 短剧（≤3min/集）每集 1 个核心情节点
3. 长篇连续剧每集 2-3 个情节点，结尾留钩子
4. 每幕 2-5 个节拍（beat），节拍是推动剧情的最小单元
