# 改编策略

## 输入
- 故事骨架（StorySkeleton）
- 原文文本

## 输出格式
结构化 JSON，包含以下字段：

- **sourceType**：原文类型（novel/outline/script）
- **tone**：调性（如"轻松搞笑""悬疑紧张""温情治愈"）
- **pacing**：节奏（fast/medium/slow）
- **omitRules**：删减规则列表（如"删内心独白""合并配角"）
- **emphasis**：强调要素列表（如"强化第一集钩子""保留高燃台词"）

## 改编原则
1. 保留核心冲突和人物弧光
2. 叙述转为可拍摄场景
3. 内心独白转为表情/动作或画外音
4. 长段落拆为多场短戏（每场 30-60s）
5. 角色语言风格保持一致
