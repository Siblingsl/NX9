NX9 AI 漫剧工作流产品重构需求（Product Refactor PRD）
一、项目背景

当前 NX9 已基本完成 AI 漫剧工作流的大部分基础能力，包括：

剧本生成
文本切分
分镜拆解
角色配置
场景配置
Prompt 生成
图片生成
视频生成
运镜
配音
导出

从工程角度来看，模块已经较完整。

但是从产品体验来看，目前产品存在大量问题。

这些问题不是 Bug，而是整个产品交互逻辑存在严重问题，导致用户第一次进入后不知道应该做什么，也不知道当前完成到了哪里，因此整体体验非常差。

本次优化目标不是增加新的 AI 能力，而是重新整理整个产品交互流程，使其真正成为一款可以正常创作 AI 漫剧的产品。

四、技术重构要求（重点）

除了产品体验，还需要对底层架构同步重构，避免后续继续堆积技术债。

Workflow 数据驱动

禁止将步骤、节点、模式、连线写死在前端组件中。

建立统一的 Workflow Schema，例如：

workflowId
mode
steps
nodes
edges
dependencies
outputs
validations
status

所有流程均由配置驱动，前端负责渲染，后端负责状态。

全局状态机（State Machine）

建立统一的项目状态管理：

Project Status（项目级）
Workflow Status（流程级）
Step Status（步骤级）
Node Status（节点级）
Task Status（任务级）

所有状态必须来源唯一（Single Source of Truth），禁止多个组件各自维护状态导致不一致。

Node 输入/输出规范

每个节点必须有统一的数据契约：

Output
Validation
Execute
Retry
Rollback（可选）

禁止节点之间直接读取彼此内部状态，应通过统一的数据流传递。

五、重构原则（必须遵守）
先完成一个完整闭环，再扩展功能。
优先保证“剧本 → 分镜 → 图片 → 视频 → 导出”能够稳定完成，而不是继续增加新节点。
产品优先于技术。
每一个界面、按钮和流程都应回答一个问题：用户为什么需要它？如果不能回答，就考虑删除或隐藏。
渐进式展示。
默认界面只展示当前任务所需的信息，高级功能通过折叠、侧栏或高级设置进入。
配置驱动，而非硬编码。
所有模式、步骤、节点、连线都应可通过配置扩展，避免未来新增模式需要修改大量代码。
每一步都有结果反馈。
用户在任何关键操作后，都应立即看到成果（角色卡、图片、视频、时间轴、进度等），持续获得创作成就感。

最后，我建议你再补一章（我认为价值最高）
产品设计红线（Product Constitution）

这不是需求，而是整个项目以后任何人（包括 Cursor）都不能违反的宪法。

例如：

🚫 永远不要
不要为了增加功能增加按钮。
不要为了展示能力默认展示高级选项。
不要让用户填写 AI 能推断的信息。
不要出现超过三级操作路径。
不要出现两个入口完成同一件事。
不要出现页面职责重叠。
不要为了技术实现暴露技术概念。
不要让用户等待时没有反馈。
不要让用户自己寻找下一步。
不要在一个页面完成两种不同任务。
✅ 永远坚持
一个页面一个核心任务。
一个步骤一个明确目标。
AI 优先，用户确认。
默认简单，按需高级。
自动化优先于配置化。
配置化优先于硬编码。
每一步都有即时反馈。
每一步都可以中断、恢复、重试。
所有状态都有明确来源。
所有流程都可以回溯。

任何修改都必须以"降低用户学习成本"为最高优先级，而不是增加功能。

具体来说：

能自动完成的，就不要让用户配置。
能推断的，就不要让用户填写。
能隐藏的高级功能，就不要默认展示。
能一键完成的，就不要拆成多个步骤。
任何新增功能都必须回答一个问题："第一次使用的用户，30 秒内知道它是干什么的吗？" 如果答案是否定的，就重新设计。
