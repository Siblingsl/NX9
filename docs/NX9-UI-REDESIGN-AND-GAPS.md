> **文档性质**：对照 `docs/NX9-FEATURE-DESIGN-LOG.md` 的**未完成/半落地功能清单**，以及**编剧台弹窗**、**导演台卡片+弹窗**的推倒重来原型与 UI 规格。  
> **查档日期**：2026-07-21（二次复核）  
> **口径**：只写「未落地 / 半落地 / 需重做 UI」事实；**不使用完成度百分比**。  
> **硬约束**：
> 1. 功能行为与数据契约**必须保留**（见各章「功能保留清单」）；只推倒原型与视觉壳。  
> 2. 交互壳仍为 **ScreenModal**（弹窗节点不准改壳类型）。  
> 3. 视觉对齐画布 Desk 族：`desk-palette.css` 的 `--desk-*`（炭黑/暖纸 + 暖金），禁止青光 HUD、扫描线、荧光 pill 堆、多套互不统属的局部色板。  
> 4. 与 `NX9-FEATURE-DESIGN-LOG.md` 并存；本文是**缺口 + UI 重设计**实施依据，不覆盖设计日志正文。

## 0. 设计原则（两台共用）

### 0.1 一句话目标

**美观 · 简洁 · 大气**：一眼只读一件事；次要操作收进次级面；留白与字重建立秩序，不用装饰堆「高级感」。

### 0.2 Desk 视觉系统（强制）

| Token | 用途 |
|-------|------|
| `--desk-bg` / `--desk-bg-2` / `--desk-bg-3` | 底 / 面板 / 抬升面 |
| `--desk-ink` / `--desk-muted` / `--desk-faint` | 主文 / 次文 / 辅助 |
| `--desk-line` / `--desk-line-strong` | 分割 |
| `--desk-accent` / `--desk-accent-soft` / `--desk-accent-line` | 唯一强调色（暖金） |
| `--desk-ok` / `--desk-warn` / `--desk-run` | 态 |
| `--desk-shadow` / `--desk-card-bg` / `--desk-media-bg` / `--desk-stage-bg` | 卡 / 媒区 / 舞台 |

### 0.3 布局戒律

1. **一屏一主任务**：编剧台主任务 = 共创/确认成稿；导演台主任务 = 批出关键帧或摆 3D（按 Tab）。  
2. **主 CTA ≤ 2**：实心主按钮 + 幽灵次按钮；其余进「更多」。  
3. **禁止芯片丛林**：策略/锁/筛选默认折叠为「设置」抽屉，展开后仍用列表行，不用一排 8 个 pill。  
4. **字号阶梯**：眉题 10 / 标题 16–18 / 正文 13 / 辅助 11；禁止满屏 9–10px 工程字。  
5. **圆角**：卡 14–16、控件 10、Modal 内面板 12；统一，不混用 6/20。  
6. **图标**：细线 1.5–1.75 stroke；同屏图标尺寸一致；不为装饰加图标。  
7. **动效**：仅开台淡入、Tab 切换、进度条宽度；禁止呼吸灯满屏。

### 0.4 明确不做

- 不做第三套 NLE / 不做非 ScreenModal 壳  
- 不把功能删掉「为了好看」  
- 不引入新的品牌色（紫/霓虹/纯黑赛博）  
- 不在摘要卡堆可编辑表单

# 2. 编剧台 · 弹窗原型与 UI 推倒重做（`script-desk`）

## 2.1 现状问题（查档结论）

| 问题 | 表现 |
|------|------|
| 信息架构乱 | 左栏同时塞技能芯片 + 快捷上传/抽取/三套导出/清会话；中 Agent；右成稿/Bible/诊断 —— 三栏同权重，无呼吸 |
| 与画布不搭 | 控件密度像后台表单；字号偏小；按钮条碎片化；缺少 desk 卡片的「媒区 + 一句状态」秩序 |
| 主任务不清 | 「确认成稿」与导出、抽取、技能抢视线 |
| 技能呈现差 | 竖列芯片难扫读，像工具箱而非创作步骤 |

**约束**：`kind=script-desk`、ScreenModal、`ScreenplayPackage` / Agent / 上传 / Bible / 确认 / 导出 / Dev Pack **功能全保留**。

## 2.2 产品定位（不变）

编剧台 = **叙事成稿 Agent 台**：选题→世界观/人物→剧情节奏对白爆点→生成/上传成稿→抽取 Bible draft→确认成稿。  
**不负责拆镜**（分镜台）。

## 2.3 画布摘要卡（可轻改，非本次主诉）

摘要卡保持简洁即可（非弹窗重点）：

```
┌──────────────────────────────────────┐
│ 编剧台 · 成稿                          │
│ 《标题》                    [已确认/草稿] │
│ 3 集 · Bible 角 4 · 场 2               │
│ 一句 logline…                          │
│                         [打开编剧台]    │
└──────────────────────────────────────┘
```

- 无芯片排、无多按钮；整卡点击开 Modal。  
- 状态色：确认=`--desk-ok`，草稿=`--desk-muted`，运行=`--desk-run`。

## 2.4 弹窗信息架构（新）

**废除**现行左-中-右三等分永久并排。

改为：**顶栏 + 主舞台（对话/上传）+ 可折叠右侧「成稿稿纸」+ 底栏主操作**。

```
┌─ ScreenModal · 编剧台  width: min(1120px, 100vw-40px) ─────────────────┐
│ [编剧台]  《标题可编辑》     状态·草稿/已确认     [成稿稿纸 ▸] [⋯更多] [×] │
│ 副文：Agent 共创成稿 · Bible 叙事稿 · 确认后交设定检查/分镜台              │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   ┌─ 技能轨（单行横滑，非左栏堆）─────────────────────────────────────┐ │
│   │ 选题  世界  人物  剧情  节奏  对白  爆点  一致  │生成│  · 上传成稿 │ │
│   └──────────────────────────────────────────────────────────────────┘ │
│                                                                        │
│   ┌─ 主舞台 ─────────────────────────────────────────────────────────┐ │
│   │                                                                  │ │
│   │   【Agent 模式】消息流（大留白、气泡层级靠左边线+字重）              │ │
│   │   待应用产出：卡片内 [应用此步]（唯一强 CTA 于消息内）               │ │
│   │                                                                  │ │
│   │   【上传模式】居中投放区 + 大文本；无侧栏噪音                       │ │
│   │                                                                  │ │
│   │   输入条：多行 · @ 提及 · 发送                                    │ │
│   └──────────────────────────────────────────────────────────────────┘ │
│                                                                        │
├────────────────────────────────────────────────────────────────────────┤

**右侧「成稿稿纸」Drawer**（默认展开宽 360；可收起只留 ▸）：

```
│ 成稿 │ Bible │ 诊断 │
│──────┴───────┴──────│
│ 分集折叠 · 正文编辑   │

### 2.4.1 模式切换

| 模式 | 进入 | 主舞台 |
|------|------|--------|
| Agent | 点技能或默认 | 对话 |
| 上传 | 技能轨最右「上传成稿」或 ⋯ | 投放区 |
| 稿纸 | 顶栏「成稿稿纸」 | 不换主舞台，仅开 Drawer |

### 2.4.2 「⋯更多」菜单（收纳）

- 导出 MD / JSON / ZIP  
- 清会话  
- Dev Prompt Pack（双闸可见）  
- 续写指定集（或放在稿纸分集行内）

**禁止**把三套导出永久钉在左栏。

## 2.5 UI 规格（新）

### 2.5.1 Modal 壳

| 项 | 规格 |
|----|------|
| variant | 与现 ScreenModal 一致；背景遮罩沿用 stage 深半透 |
| 面板底 | `--desk-bg`；内部分割 `--desk-line` |
| 顶栏高 | 56–64px；标题 17px / 650；副题 `--desk-muted` 12px |
| 底栏高 | 56px；主 CTA 右对齐，高 36–40，圆角 10，底 `--desk-accent`，字 `--desk-btn-primary-ink` |

### 2.5.2 技能轨

- 高度 40；项为文字按钮，选中：底 `--desk-accent-soft` + 字色 ink，**不是**实心金块满轨。  
- 「生成」与「上传」用细竖线与创作技能分隔。  
- 当前技能名可在输入条上方一行淡字：`正在：人物构建`。

### 2.5.3 Agent 消息

| 角色 | 样式 |
|------|------|
| system | 居中淡字，无气泡 |
| assistant | 左对齐；左边线 2px `--desk-accent-line`；底 `--desk-panel`；圆角 12 |
| user | 右对齐；底 `--desk-bg-3` |
| 待应用 | 消息下一条带边框的「产出待确认」条，仅 `[应用此步产出]` |

### 2.5.4 输入条

- 圆角 12；边框 `--desk-line-strong`；focus 时 `--desk-accent-line`。  
- `@` 下拉：分组「Bible draft」「素材库」；库项可带 `锁定` 小标（功能保留，样式克制）。

### 2.5.5 稿纸 Drawer

- 分集：大标题 + textarea，行高 1.55；禁止表格感。  
- Bible 卡：名 14 粗 + 两行 meta + draft 角标；无「健康度仪表盘」。  
- 爆点轨：单行细轨 + 圆点；高度 ≤ 28。

### 2.5.6 动效

- Drawer 滑入 180ms ease。  
- 发送后输入清空；busy 时主 CTA disabled + 轨上小 spinner（非全屏遮罩）。

## 2.6 功能保留清单（重做时禁止丢失）

| 能力 | 落点（新 UI） |
|------|----------------|
| 技能：topic/world/character/plot/pacing/dialogue/hooks/consistency/generate | 技能轨 |
| 上传/粘贴成稿 | 上传模式 |
| 抽取 Bible | 底栏幽灵 |
| 确认成稿 / 取消确认语义 | 底栏主 CTA |
| 应用此步产出 | 消息内 |
| Agent 会话持久化 | 不变 |
| 续写单集 | 稿纸分集行或 ⋯ |
| 导出 MD/JSON/ZIP | ⋯更多 |
| `@` Bible + 素材库 | 输入条 |
| 爆点轨、诊断 | 稿纸 Tab |
| legacy 镜表提示 | 底栏或稿纸顶一条 warn |
| Dev Prompt Pack | ⋯ + 双闸 |

## 2.7 实现注记

- CSS：新建 `script-desk.v2.css`（或重写 `script-desk.css`），类前缀建议 `.sd2-`，避免与旧类混用。  
- 结构可保留同一 `ScriptDeskBlock.tsx` 状态机，只换 JSX 布局。  
- **禁止**为 UI 改 `ScreenplayPackage` 字段名。

## 2.8 验收（UI）

1. 开台后 3 秒内能说清：我在和 Agent 共创，右侧是成稿。  
2. 主 CTA「确认成稿」在任何滚动位置底栏可见。  
3. 导出不出现在首屏主视觉。  
4. 浅色/深色主题均只用 `--desk-*`，无硬编码荧光色。  
5. 功能保留清单逐项手测通过。

# 3. 导演台 · 卡片与弹窗原型与 UI 推倒重做（`director-desk`）

## 3.1 现状问题（查档结论）

| 问题 | 表现 |
|------|------|
| 摘要卡杂乱 | 媒区 + 文案 + 大数字 + stats 行 + chips 行 + trail + 底栏双按钮；画布上像仪表盘碎片 |
| Modal 不可用感 | 「生产」Tab 纵向堆：进度、筛选、大量 checkbox、并发重试、风格 seed、队列、预览 Tab、底栏动作 —— 同屏多主任务 |
| 无美感 | 9–10px 字、控件挤、预览区被压矮；3D 与生产视觉语言不统一 |

**约束**：ScreenModal；批出/队列/锁/3D 嵌入/截图写回/送出/审阅/预览切换/沉浸/机位预设/Dev 字段 **功能全保留**，仅重排与换肤。

## 3.2 产品定位（不变）

导演台 = **3D 机位参考（可选）+ 本集关键帧批出 + 审阅/送视频**。  
分镜构图确认归分镜台；成片编排归智能剪辑。

## 3.3 画布摘要卡（推倒）

**只保留 4 层信息**：身份 · 一句状态 · 双指标 · 双 CTA。

```
┌─────────────────────────────────────────────┐
│ ┌─────────┐  导演台 · 本集                    │
│ │         │  关键帧导演工作台                     │
│ │  媒区   │  已出 8/12 · 3D 5                 │
│ │ 16:9    │  ████████░░░░  67%                │
│ └─────────┘  （运行中则替换为「批出中 3/12」）   │
│                                             │
│  [批出未完成]              [开台]             │
└─────────────────────────────────────────────┘
```

### 3.3.1 卡面规则

| 元素 | 规则 |
|------|------|
| 媒区 | 固定比例约 4:3 或 16:10；`previewUrl` → 末次关键帧 → 3D capture → Clapperboard 空态（`--desk-media-bg`） |
| 标题 | 「关键帧导演工作台」一行；眉题「导演台 · 本集」 |
| 指标 | **仅两数**：已出 `withFrame/total`、3D `with3d`（无 3D 时可藏） |
| 进度条 | 单条，高 3–4px，`--desk-accent` |
| 芯片 | **卡片上禁止**「角色锁·场景锁·优先3D」等；策略只在 Modal |
| CTA | 幽灵「批出未完成/停止」+ 实心「开台」；宽卡底左右分布 |
| 错误 | 卡下最多一行 `--desk-warn`，不进卡内堆 |

尺寸：建议卡内容区宽与现 BlockShell 一致；高度目标 **≤ 现卡约 70%**，去掉 chips/stats 网格。

## 3.4 ScreenModal 信息架构（推倒）

**左队列固定 + 右工作面按模式切换**；生产设置**永不与大预览抢同一视线**。

### 3.4.1 「批出设置」抽屉（生产 Tab 专用）

点击「批出设置」自底或右侧滑出 **单列表单**，关闭后回到大预览：

- 跳过已有 / 跳过已审  
- 角色锁 / 场景锁 / 风格锁 / 优先 3D  
- 并发、重试  
- 风格 prompt / seed / 同步到 picture-gen  
- 批完自动开审阅  

**所有现 checkbox 迁入此处**；生产主面不再出现勾选丛林。

### 3.4.2 队列行

```
[thumb 40]  #03  待出    有3D   ›
            场景名 · 一句动作
```

- 选中：左边线 accent + 底 soft。  
- 「进 3D」：行尾图标按钮，切 Tab=3D 并 focus 该镜。  
- 筛选：顶上 **一个** select/segment，不要 5 个 chip 横排。

### 3.4.3 3D 沉浸

- 「沉浸」：隐藏左队列与顶副文，舞台近全屏；顶留「退出沉浸」。  
- 仍同一 Modal，不新节点。

### 3.4.4 送出 Tab

三张等高卡片，每卡：标题 + 一句说明 + 一个按钮。禁止再堆 6 个同级 `dd-btn`。

- CSS：`director-desk.v2.css` 或重写；前缀 `.dd2-`。

## 3.8 验收（UI）

1. 画布卡 1 秒可读：出了多少帧、能不能开台。  
2. 生产 Tab 首屏：**大图 + 一条操作**；设置默认收起。  
3. 勾选策略不出现在首屏。  
4. 3D Tab 进入后舞台占主导；沉浸可用。  
5. 送出 Tab 三个动作互不抢。  
6. 功能保留清单手测全过；批出结果与重做前一致。
