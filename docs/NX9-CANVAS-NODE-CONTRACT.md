# NX9 画布节点设计契约

> 状态：生效中 · 高级画布（全屏舞台）专用  
> 原则：黑舞台 + 统一浅色摘要卡 + 二级面分流 · 禁止第三套样式

---

## 1. 三层结构（必须遵守）

| 层 | 是什么 | 样式 | 禁止 |
|----|--------|------|------|
| **L0 画布壳** | 底岛 / 顶条 / 步骤条 | 深色玻璃 | 白底 IDE 顶栏/左坞 |
| **L1 摘要卡** | 画布上每一个节点 | **唯一**浅色节点卡 | 卡上堆完整长表单 |
| **L2 二级面** | 细节编辑 | 生成→**底部工作区**；设定/拆分/编辑→**屏幕弹窗** | 节点旁锚定小浮层 |

底部工作区（Attached Workspace）**不得**改为屏幕弹窗。

---

## 2. L1 摘要卡骨架（所有 kind 同一模板）

```
┌────────────────────────────┐
│ 色条 · 标题（壳层）  · 状态 │  ← BlockShell 负责
├────────────────────────────┤
│  主视觉区（固定 16:10）      │  ← 图 / 视频 / 空态
│  可选：最多 3 个数字 stat    │
│  一行摘要（截断）            │
│  [次要] …        [主操作]   │
└────────────────────────────┘
```

- **宽度**：280px（统一）
- **圆角 / 阴影 / 字阶**：仅 `node-stage-card.css` + `BlockShell`
- **主操作**：语义化（运行 / 拆分 / 检查 / 编辑），青绿实心
- **次要**：幽灵按钮，最多 2 个
- **列表过长**：摘要卡只显示前 3～5 条或「共 N 项」，完整进 L2

组件入口：`blocks/shared/NodeSummaryBody.tsx`

---

## 3. L2 二级面分流

| 节点类 | 打开方式 | 容器 |
|--------|----------|------|
| 生成 / 预览 / 工具（prompt、picture-gen、clip-gen、storyboard-preview…） | 选中节点自动 / 点展开 | **底部跟随工作区** |
| 设定 / 拆分 / 分镜编辑 / 导演控制 | 点主操作或列表项 | **ScreenModal** |
| 轻量确认 | 卡上主按钮直接执行 | 无二级面 |

弹窗皮肤：`ScreenModal` + `stage-bible.css`（浅色高对比表单）。

---

## 4. 材质与配色

| 元素 | 规范 |
|------|------|
| 画布底 | `#0c0e12` 深色舞台 |
| 节点卡 | 浅色纸 `#fff → #f7f5f1`，深色字 `#1a1814` |
| 控制台浮层 | 深色玻璃（底岛） |
| 强调色 | 青绿 `#0d9488` / `#2dd4bf` |
| 禁止 | 浅色卡上浅色字；卡内再套第三套深色 HUD |

---

## 5. 新增节点 checklist

1. 登记 `attached-workspace`（二级面类型）
2. 画布渲染只用 `BlockShell` + `NodeSummaryBody`（或 `CanvasNodeShell`）
3. 完整编辑进底部工作区或 ScreenModal，**不**写节点旁 absolute 浮层
4. 不引入新的局部 CSS 主题文件（扩展现有 token）

---

## 6. 与历史代码关系

- `CanvasNodeShell` / `CanvasNodeBody`：生成类摘要标准实现  
- `stage-bible.css`：设定系弹窗与摘要辅助样式（须保持深字浅底）  
- `shouldUseCompactNodeShell`：为 true 时走摘要卡，禁止回退完整表单卡  

修订本文档须同步改实现，禁止文档与代码长期分叉。
